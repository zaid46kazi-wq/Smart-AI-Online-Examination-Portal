💡 HOW IT WORKS

1. GZIP Compression
   Client Request → Server (with compression middleware) → 
   Response compressed (40-60% smaller) → Client decompresses

2. Rate Limiting
   Request → Check IP & request count → 
   If OK → Process → If limit exceeded → 429 (Too Many Requests)

3. Batch Inserts
   Individual records → Format as array → Single batch insert → 
   All records inserted in one DB call (95% faster)

4. Static Caching
   First visit → Server sends files + cache headers (1 day) → 
   Return visits → Browser uses cached files (instant)

5. API Caching
   First request → Server computes + sends cache header → 
   Repeat requests within 5min → Browser returns cached response
